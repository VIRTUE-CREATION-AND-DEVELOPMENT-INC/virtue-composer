# @virtuecreation/composer-registry

Machine-readable component, composition, and page-blueprint contracts for Virtue Composer tooling.

Components are runtime contracts. Compositions are copyable project-owned JSX and CSS
wireframes selected through natural-language metadata. Blueprints arrange compositions
into adaptable page baselines.

Decision-grade component records add:

- native and complex alternatives, state ownership, server/client boundaries,
  serialization, focus ownership, failure modes, and composition relationships;
- measured own-module runtime cost, engine, effect, listener, portal, scale,
  and lazy-load guidance;
- practical HTML, data-sensitivity, validation, secret, network, persistence,
  redirect, and commerce trust boundaries.

`stability-evidence.json` is a versioned local evidence registry. It
distinguishes production adoption from maintained fixtures and stores manual
accessibility gaps, defects, revisions, unresolved risks, and reviewer
decisions without auto-promoting components.

Measured module bytes describe the component's own unminified ESM output. They
exclude dependencies and consumer bundler behavior.
