# Virtue Composer 0.7.0 launch readiness

Date: July 24, 2026

Release: `0.7.0`

Status: approved registry-foundation release

## Release scope

Version 0.7.0 is an additive registry and CLI release. It adds decision-grade
selection guidance for an initial set of high-risk components, measured
own-module runtime profiles, practical security and trust-boundary metadata,
metadata-driven Doctor warnings, and a versioned local stability-evidence
report. It does not change stable 0.6 component APIs, replace customized
wrappers, install a visual theme, or claim completion of the later page-plan,
lifecycle, wrapper-drift, browser-harness, and benchmark roadmap.

## Release evidence

- `npm run release:check` passed from the release tree.
- Registry validation covered 128 components, 34 compositions, three
  blueprints, and two evidence sources.
- 136 unit tests, lint, type checking, Composer and showcase production builds,
  runtime budgets, and strict Doctor passed.
- Strict Doctor parsed 173 of 173 showcase source files with no errors or
  warnings.
- Playwright completed 70 Chromium, Firefox, and WebKit checks with two
  intentional non-Chromium visual-baseline skips.
- All 121 published 0.4 export subpaths and peer dependencies remain.
- All four package dry-runs and the packed Composer and CLI consumers passed.

## Security accounting

`npm audit --omit=dev` reports the existing three high-severity PostCSS and
Sharp findings under the private Next.js showcase dependency graph. npm's
suggested force remediation is an invalid downgrade to Next.js 9.3.3. These
dependencies are not bundled in the four public package tarballs, so the
documented vendor exception from 0.6.0 remains in force.

## Publication and rollback

Publish in dependency order under `next`: registry, Composer, ESLint config,
then CLI. Run the public npm smoke before promoting the four exact 0.7.0
versions to `latest`.

Published npm versions are immutable. If the public smoke fails, leave
`latest` on 0.6.0 and publish a corrected patch. Consumers can roll back by
pinning Composer and the CLI to `0.6.0` and restoring their lockfile; existing
project-owned wrappers and styles do not need to be replaced.
