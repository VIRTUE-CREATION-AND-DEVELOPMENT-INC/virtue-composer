# Virtue Composer 0.7 implementation record

Status: released July 24, 2026. This document records the compatible
registry-foundation scope shipped in 0.7.0 and the broader roadmap that remains;
the release does not claim that the full agentic planning system is complete.

## Implemented Stage A foundation

- Twelve high-risk or selection-sensitive components now expose validated
  decision guidance: `Dialog`, `Form`, `FileUpload`, `DataTable`, `DataGrid`,
  `RichTextEditor`, `MarkdownEditor`, `MediaPicker`, `ImportPanel`,
  `HoneypotField`, `CartSummary`, and `DiscountCodeInput`.
- Their runtime profiles declare client requirements, engines, own-module byte
  measurements, complexity, effects, listener and portal characteristics,
  simpler alternatives, scale guidance, and lazy-load suitability.
- Build analysis reports every profiled component and rejects an effect-count
  mismatch or an own-module measurement drift greater than ten percent.
- Forms, uploads, editors, media, imports, abuse signals, and commerce summaries
  expose trust-boundary metadata that never treats frontend validation as a
  security boundary.
- `FileUpload` contributes the first metadata-driven trust Doctor rule:
  projects should declare an `accept` policy and must repeat type, size, and
  content validation on the server. New 0.7 configs enable the warning;
  undeclared 0.6 configs retain an implicit off default until they opt in.
- The versioned stability-evidence model records BGR as one production Adopt
  source and the Composer showcase as a maintained Bootstrap fixture. Manual
  screen-reader, physical-touch, and Windows high-contrast evidence remains
  explicitly unrecorded.
- `virtue-composer stability` reports evidence and human-review gates per
  component. No count can auto-promote a component.

Measured bytes are unminified ESM for each component's own TypeScript-compiled
module. They exclude dependencies, tree-shaking, minification, compression,
framework chunks, and consumer bundler behavior.

## Follow-on Stage A coverage

- Expand decision, runtime, and trust metadata from the initial prioritized set
  to every relevant ambiguous component and composition.
- Add richer negative Doctor fixtures for untrusted links, rendered content,
  media URLs, admin authority, and commerce boundaries where static source
  checks can be accurate without noisy compliance theatre.
- Record component-specific defects and real manual accessibility sessions.
- Add a second meaningfully different production adoption before using project
  diversity as stability evidence.

## Sequenced continuation

1. Add the versioned page-plan, design-context, and local adaptation-evidence
   contracts with deterministic validate, inspect, migrate, and dry-run apply.
2. Formalize Bootstrap, Adopt, and Evolve defaults, then add wrapper provenance
   and drift classification without overwriting customization.
3. Add frozen positive and rejection planning fixtures, the Next.js-first
   browser verification harness, and packed-consumer verification.
4. Run and publish the paired Composer/control benchmark with frozen inputs and
   transparent raw results.
5. Complete migration and rollback docs before releasing the workflows that
   require them.

The separate 0.6.1 maintenance candidates remain intentionally outside this
0.7 implementation slice.

## Release validation

The final release commit passed `npm run release:check`: registry and contract
validation, type checking, 136 unit tests, lint, production builds, runtime
budgets, strict Doctor, fresh/root/upgrade smokes, 70 browser checks with two
intentional cross-browser visual-baseline skips, published-export
compatibility, package inspection, and packed Composer and CLI consumer builds.

Rollback is package-level because 0.7.0 makes no breaking stable API or
generated-wrapper replacement: consumers can pin Composer and the CLI to
`0.6.0`, restore their lockfile, and retain their project-owned wrappers and
styles. New trust-boundary enforcement is explicit in generated 0.7 configs;
legacy configs without that key retain the 0.6 off default.
