# Virtue Composer 0.6.0 launch readiness

Date: July 23, 2026  
Release: `0.6.0`  
Status: ready for release commit; publication requires explicit npm authorization

## Launch decision

Version 0.6.0 is ready to publish as the next minor release. It expands the canonical contract from 120 to 128 components and adds the first project-owned composition layer: 34 composition wireframes across a core catalog and five specialized packs, three page blueprints, natural-language discovery, ejectable CLI installation, and a generated browser sandbox.

The release also includes the component stabilization, candidate-selection enforcement, accessibility work, and bundle budgets described in `STABILIZATION-REPORT.md`. No package was published and no dist-tag was changed during preparation.

## Included scope

New experimental components:

- SelectableCardGroup
- ResizablePanels
- Wizard
- TransferList
- ColorPicker
- RatingInput
- MentionInput
- EditableList

Composition catalog:

- FAQ: split accordion and stacked list
- Services: icon grid and media grid
- Process: numbered steps and alternating media timeline
- Proof: metric strip, editorial testimonials, and partner logo wall
- Collections: featured gallery, featured-plus-grid stories, people profiles, and featured event list
- Conversion: inline newsletter and dismissible announcement
- Contact and participation: pathways and location/contact cards
- Contact: a split inquiry form with direct contact routes
- General-purpose content: reversible text/media split and stacked featured-media sections
- Completion: confirmation with next actions
- Commerce pack: product collection, product detail gallery, and cart/checkout summary
- Guided-workflows pack: multi-step selection, estimate, and booking flows
- Search pack: filtered results with a master-detail layout
- Application pack: authentication shell, dashboard overview, resource index, and resource editor detail
- Immersive pack: horizontal story and chaptered presentation

Page blueprints:

- Service business
- Community/nonprofit
- Editorial/portfolio

Composition JSX and CSS are copied into the consuming project and become project-owned. Composer supplies the structural and behavioral primitives; it does not introduce a rigid runtime composition abstraction or own the final visual design.

## Security accounting

`npm audit --omit=dev` reports three high-severity findings in the private showcase/tooling dependency graph: PostCSS advisories under Next.js and Sharp/libvips advisories. npm's automated remediation proposes Next.js 9.3.3, which is an incompatible downgrade from Next.js 16.2.11.

This is a documented vendor exception for the release candidate:

- the affected packages enter through the private Next.js showcase;
- Composer, registry, CLI, and ESLint tarballs do not bundle Next.js, PostCSS, or Sharp;
- Composer declares Next.js as a peer dependency, so consuming applications own their framework version and deployment exposure;
- no compatible upgrade is currently offered by `npm audit`.

If project policy prohibits an unresolved high advisory even when it is confined to private tooling and has no compatible vendor fix, publication must wait for an upstream release.

## Publication checklist

These release-operator actions remain intentionally unperformed:

1. Review the complete uncommitted 0.6.0 batch.
2. Commit and tag the approved release.
3. Re-run `npm run release:check` and `npm audit --omit=dev` from the release commit.
4. Publish the four packages under the `next` dist-tag in dependency order: registry, Composer, ESLint config, CLI.
5. Run `npm run smoke:npm-release` from the release commit and verify release notes, composition discovery, and composition copying from a separate empty project.
6. Promote all four packages to the `latest` dist-tag only after the published-artifact checks pass.

The release operator can use this sequence after the release commit is on
`main`:

```sh
npm run release:check
npm audit --omit=dev
git tag -a v0.6.0 -m "Virtue Composer 0.6.0"
git push origin main
git push origin v0.6.0

npm publish --workspace @virtuecreation/composer-registry --tag next
npm publish --workspace @virtuecreation/composer --tag next
npm publish --workspace @virtuecreation/eslint-config-composer --tag next
npm publish --workspace @virtuecreation/composer-cli --tag next

npm run smoke:npm-release

npm dist-tag add @virtuecreation/composer-registry@0.6.0 latest
npm dist-tag add @virtuecreation/composer@0.6.0 latest
npm dist-tag add @virtuecreation/eslint-config-composer@0.6.0 latest
npm dist-tag add @virtuecreation/composer-cli@0.6.0 latest
```

If the post-publication smoke fails, leave `latest` unchanged, correct the
release issue, and publish a new patch version. Published npm versions are
immutable and must not be overwritten.

Manual screen-reader testing, a physical touch device, and Windows High Contrast remain recommended evidence before promoting experimental components or compositions to a more mature stability level.
